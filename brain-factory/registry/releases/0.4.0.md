# Framework release 0.4.0

- Date: 2026-07-06
- Manifest compatibility: `>= 1.0.0`

## Summary

This release codifies proven Acme onboarding learnings into reusable framework
contracts for inspect-first onboarding and queue operation.

## Core modules

| Module | Change | Notes |
| --- | --- | --- |
| `core-commands` | changed | Inspect-first onboarding contract now requires explicit runtime-vs-doc drift checks before apply. |
| `continuity-log` | changed | No-loss migration contract now requires byte-exact verification for non-text artifact migration. |
| `decision-board` | changed | Queue state model now supports explicit operator hold markers to pause auto-flow after out-of-band completion. |
| `operating-contract` | unchanged | |
| `session-ritual-hooks` | unchanged | |
| `capabilities-map` | unchanged | |
| `docs-mesh` | unchanged | |

## Action for brains

1. Run `<prefix>-upgrade` (dry-run, then `--apply`) to pick up codified
   onboarding/queue contracts in the core layer.
2. For inspect-first onboarding, enforce:
   - non-text migration through base64 transport with byte-exact verification,
   - runtime-vs-doc drift reconciliation before apply.
3. For queue reconciliation after out-of-band completion, mark the completed task
   `done` and keep downstream tasks `blocked` only when an explicit
   `hold_reason` is recorded.
