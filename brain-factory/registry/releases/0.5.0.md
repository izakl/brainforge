# Framework release 0.5.0

- Date: 2026-07-06
- Manifest compatibility: `>= 1.0.0`

## Summary

This release codifies two permanent operating standards as inherited core
policy across all lanes and both agent toolchains:

1. **SYNC-LATEST-FIRST** — always sync to current online default branch before
   acting.
2. **CLEANUP-NO-STALE-STATE** — remove stale branches/worktrees/sessions with a
   no-loss gate before deletion.

## Core modules

| Module | Change | Notes |

| `operating-contract` | changed | Permanent standards are now codified in root and template instruction surfaces for both Copilot and Claude (`AGENTS.md`, `.github/copilot-instructions.md`, `CLAUDE.md`). |

## Action for brains

1. Run `<prefix>-upgrade` (dry-run, then `--apply`) to inherit the standards in
   all core instruction files.
2. Confirm each lane exposes both standards in:
   - `AGENTS.md`
   - `.github/copilot-instructions.md`
   - `CLAUDE.md`
3. Enforce no-loss cleanup before branch/worktree/session deletion and keep
   stale-state probes active.
