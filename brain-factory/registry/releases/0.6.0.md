# Framework release 0.6.0

- Date: 2026-07-06
- Manifest compatibility: `>= 1.0.0`

## Summary

This release codifies a third permanent operating standard:
**CONTINUITY-CAPTURE / BRAIN-MEMORY WRITEBACK**.
Lane work in any lane-owned repo (product/runtime or governance) must be
captured in the lane brain continuity system with WHAT/WHY/WHERE/OUTCOME.

## Core modules

| Module | Change | Notes |
| --- | --- | --- |
| `operating-contract` | changed | Adds continuity-capture / brain-memory writeback standard to hub and template instruction surfaces (`AGENTS.md`, `.github/copilot-instructions.md`, `CLAUDE.md`). |
| `continuity-log` | changed | Makes cross-repo continuity writeback timing explicit (in-progress at start/PR open, finalized on merge). |

## Action for brains

1. Run `<prefix>-upgrade` (dry-run, then `--apply`) to inherit continuity-capture
   language across all instruction surfaces.
2. Ensure each lane records continuity for work across product/runtime and
   governance repos with:
   - WHAT
   - WHY (trigger/feedback)
   - WHERE (repo + PR/commit)
   - OUTCOME (validation + merge status)
3. Enforce timing:
   - capture in-progress at start / PR open
   - finalize on merge
4. Cross-link product/runtime PRs and continuity entries where practical.
