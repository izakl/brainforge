# Framework release 0.7.0

- Date: 2026-07-07
- Manifest compatibility: `>= 1.0.0`

## Summary

This release promotes the **QA-twin pattern** into the hub as a reusable,
inheritable core primitive. The pattern is generalized from the Northwind System
QA twin (`xs-qa`); the Northwind harness itself stays product-scoped in its own
lane. Two inherited artifacts are added:

- a new **core command** `qa` (`{prefix}-qa`) — a generic QA-twin scaffold each
  lane instantiates into its own harness;
- a new inherited **QA standard** (`04-policies/qa-standard.md`) — the QA
  discipline: stages, cadence, the FIX/IMPROVE/FLAG taxonomy, and the
  non-negotiable lessons.

## Action for brains

1. Run `<prefix>-upgrade` (dry-run, then `--apply`) to inherit the `qa` command
   and the `04-policies/qa-standard.md` standard.
2. Run `<prefix>-capabilities` to regenerate the capabilities map so it gains the
   `<prefix>-qa` row (the intent gate expects it).
3. **Instantiate the harness for this lane** — point `<prefix>-qa` at this
   product's live surfaces, source-of-truth endpoints, and scenarios, and build
   the harness under an app-repo `scripts/qa/` (or equivalent). Do **not** inherit
   another lane's harness.
4. Wire `fast` into every deploy and `deep` into the nightly scheduler.

## Notes

- Additive change: no existing command, module, or manifest field is removed.
- See ADR 0026 for the decision record (why the pattern is promoted to core and
  the harness stays per-lane).
